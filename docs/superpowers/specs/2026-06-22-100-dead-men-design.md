# TASK_SPEC.md — 100 Dead Men

## Ціль і бажаний результат
Односторінковий браузерний проєкт (index.html) — темна 8-bit симуляція "100 в'язнів і 100 скриньок". Користувач обирає стратегію, крутить міксери, запускає симуляцію та спостерігає, як в'язні один за одним відкривають скриньки в пошуках свого номера. Результат — стильний інтерактивний "той", який виглядає як проклята тюремна машина.

## Поточна поведінка і бажана
Зараз проєкту немає. Потрібен єдиний index.html із темним піксельним інтерфейсом, 10×10 сіткою скриньок, 5 стратегіями, міксерами, покроковою симуляцією, WIN/LOSE, статистикою та кумулятивним метром.

## Межі завдання (scope і non-goals)
- **Включено:** index.html, стратегії, візуалізація, статистика
- **Не включено:** npm, бекенд, білд-тули, зовнішні ресурси, звук, тести, документація API

## Обмеження і межі змін
- Один файл: `lesson2_1/index.html`
- Жодних зовнішніх залежностей
- Усі стратегії всередині файлу
- Роздільна логіка (стан, стратегії, рендеринг, контролер)

## Пакет доказів
index.html відкривається в браузері, всі 5 стратегій вибираються, симуляція працює.

## Критерії приймання
1. index.html відкривається без setup
2. Стратегії вибираються, слайдери оновлюються
3. Run/Pause/Reset працюють
4. Speed слайдер працює
5. В'язні біжать послідовно, видно скриньки
6. WIN/LOSE показується
7. Статистика оновлюється
8. Кумулятивний метр працює
9. Зміна міксерів під час роботи впливає негайно
10. **Розрив у шансах очевидний:** The Loop Protocol дає ~30%, Blind Panic ~0%
