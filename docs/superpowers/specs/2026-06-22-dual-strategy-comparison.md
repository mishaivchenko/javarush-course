# Dual Strategy Comparison — Design Spec

## Goal
Додати можливість запустити дві стратегії одночасно на двох окремих полях і порівняти результати в реальному часі.

## Scope
Один файл: `lesson2_1/index.html`. Зміни торкнуться всіх 4 секцій: CSS (layout), HTML (два поля), MODEL (два стани), RENDERER (дубльований), CONTROLLER (нова логіка).

## Approach
Архітектура: **два незалежних state + один спільний Strategies**. Кожне поле має свій `stateA` / `stateB`, свій `renderAllA()` / `renderAllB()`, свої елементи DOM. Permutation генерується **однакова для обох полів** (щоб порівняння було чесним — однаковий розклад коробок).

### Layout

```
┌─────────────────────────────────────────────────────┐
│  THE TOMB OF 100 DEAD MEN              Controls     │
├──────────────────────────┬──────────────────────────┤
│   STRATEGY A             │   STRATEGY B             │
│  ┌──┬──┬──┬──┬──┬──┐    │  ┌──┬──┬──┬──┬──┬──┐    │
│  │  │  │  │  │  │  │    │  │  │  │  │  │  │  │    │
│  │  │  │  │  │  │  │    │  │  │  │  │  │  │  │    │
│  └──┴──┴──┴──┴──┴──┘    │  └──┴──┴──┴──┴──┴──┘    │
│  Path/Status A           │  Path/Status B           │
│  Stats A                 │  Stats B                 │
├──────────────────────────┴──────────────────────────┤
│  Сравнительная статистика (Win%, AVG попыток)       │
└─────────────────────────────────────────────────────┘
```

- **Header** — title + кнопки (Run All, Pause All, Reset All) + speed slider. Без mugshot (недоречно для двох полів).
- **Two columns** — кожна колонка містить: select strategy, mixers, grid 10×10, path stripe, bottom stats
- **Comparison bar** — під обома колонками: Win%, Losses, порівняння

### State
Два окремі об'єкти: `stateA` та `stateB`. Повністю ідентична структура. Permutation однакова (`stateA.boxes === stateB.boxes` — або копія того ж масиву).

### Рендеринг
Кожен renderer дубльований: `renderBoxGridA()`, `renderBoxGridB()`, `renderPathStripeA()`, тощо. Вони ідентичні за логікою, але пишуть в різні DOM-елементи.

### Контрол
Кнопки в хедері: **Run All** (запускає обидві симуляції паралельно), **Pause All**, **Reset All**. Speed slider — спільний для обох.

### Порівняння
Під двома полями — рядок з порівняльною статистикою:

| Метрика | Стратегія A | Стратегія B |
|---|---|---|
| Win Rate | XX% | YY% |
| Avg Steps/Found | X.X | Y.Y |
| Best Streak | X | Y |

## Success Criteria
1. Два незалежних поля 10×10, кожне зі своєю стратегією
2. Обидві симуляції біжать одночасно (паралельно)
3. Однакова permutation для чесного порівняння
4. Сумісна статистика внизу
5. Всі баги попередніх змін виправлені
6. Немає скролу
7. Lovecraftian тема збережена

## Implementation Notes
- Дублювання коду renderer'ів — свідоме рішення для прозорості і простоти
- JS: `stateA` та `stateB` — копії одного шаблону
- `Strategies` об'єкт не змінюється — обидва поля використовують ті самі стратегії
- `generatePermutation()` повертає масив; `stateA.boxes = perm; stateB.boxes = [...perm]`
- Sim loop: `Promise.all([runLoopA(), runLoopB()])` для паралельного запуску
